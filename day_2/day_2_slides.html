What is a Variable? 📦

Think of a variable as a labelled box.
You give the box a name, put a value inside,
and use the name whenever you need that value.

📦
name
"Alice"
📦
age
15
📦
city
"Kathmandu"
# Create variables
name = "Alice"
age  = 15
city = "Kathmandu"

# Use them
print(name)    # Alice
print(age)     # 15

# Update them anytime!
age = 16
print(age)     # 16
🔑 = means "put this value in this box" — it's NOT an equals sign from maths!

Variable Naming Rules 📝

✅ DO THIS

  • •first_name = "Bob"
    Underscore for spaces
  • •score1 = 100
    Numbers OK — not at the start
  • •is_happy = True
    Descriptive, clear names
  • •total_price = 250
    lowercase_with_underscores (Python style)

❌ NEVER DO THIS

  • •1score = 100
    Cannot start with a number
  • •my score = 100
    Cannot have spaces
  • •my-score = 100
    Dash means subtraction!
  • •@name = "Bob"
    No special characters @ # $ %
💡 Good names describe what is INSIDE the box. student_age beats x every time — future-you will be grateful!

Strings
Type: str

Strings — Working with Text

A string is any text inside quote marks — letters, words, emoji, anything!

# Single or double quotes — both work!
greeting = "Hello, World!"
city     = 'Kathmandu'
mood     = "Python is 🔥"

# + joins strings, * repeats them
print("Hello, " + "class!")
print("⭐" * 15)
⚠️ A number inside quotes is still a string: "42" ≠ 42
7 / 21

String Superpowers 🦸

Built-in methods

word = "python"

word.upper()    # "PYTHON"
word.lower()    # "python"
word.title()    # "Python"
len(word)       # 6

F-Strings ⭐ — The Modern Way

name = "Alice"
age  = 15

# Old messy way:
"Hi, " + name + " age " + str(age)

# F-string — clean ✨
print(f"Hi, {name} age {age}")

# Maths inside curly braces!
print(f"Next year: {age + 1}")
🚀 Always prefer f-strings when mixing variables with text — so much cleaner than lots of + signs!

Numbers in Action 💡

# Shopping receipt
price    = 250
quantity = 4
discount = 50

subtotal = price * quantity
final    = subtotal - discount

print(f"Subtotal : Rs. {subtotal}")
print(f"Final    : Rs. {final}")
# input() always returns a string!
# Convert with int() for maths.

raw = input("Enter a number: ")
# raw = "42"  ← string!

num = int(raw)
print(num + 10)    # 52 ✅
print(raw + "10")  # "4210" ← strings join!
⚠️ Forgetting to convert input is the #1 beginner mistake. Always ask: "do I need maths with this?"

Booleans
Type: bool

Booleans — True or False

A boolean has only two possible values. They power every decision your program makes!

# Creating booleans directly
is_raining    = False
has_homework  = True
is_python_fun = True   # obviously 😄

# Comparison operators PRODUCE booleans
print(10 == 10)    # True   (equal to)
print(10 != 5)     # True   (not equal)
print(10 >  20)    # False  (greater than)
print(10 <= 10)    # True   (less or equal)
🧠 == checks equality  |  = assigns a value. Completely different!

type() and Type Conversion 🔎🦎

The type() Detective

print(type("Hello"))  # <class 'str'>
print(type(42))       # <class 'int'>
print(type(3.14))     # <class 'float'>
print(type(True))     # <class 'bool'>

Shapeshifting!

int("100")     # str → int   : 100
float("3.14")  # str → float : 3.14
str(42)         # int → str   : "42"
bool(0)         # 0 is always False!
bool(1)         # anything else → True
THE CLASSIC TRAP — AND THE FIX
# input() ALWAYS gives a string — even if the user types 42
age = int(input("Enter your age: "))  # ← wrap with int() immediately
print(f"Next year you'll be {age + 1}")

Lists
Type: list

Lists — Many Values, One Variable

A list stores multiple items in one place. Items are ordered and you can change them!

# Create a list with square brackets []
fruits  = ["apple", "banana", "mango"]
scores  = [95, 87, 72, 100]
mixed   = ["Alice", 15, True]   # mix types!

print(fruits)  # ['apple', 'banana', 'mango']
print(len(fruits))  # 3
📦 Think of a list as a row of numbered boxes — but the numbering starts at 0, not 1!

Tuples
Type: tuple

Tuples — Lists That Can't Change

A tuple is exactly like a list, but once created it is locked — perfect for data that should never change.

# Use round brackets () instead of []
coordinates  = (27.7172, 85.3240)  # Kathmandu GPS
rgb_red      = (255, 0, 0)
days         = ("Mon","Tue","Wed","Thu","Fri")

# Indexing works the same as lists
print(days[0])   # Mon
print(days[-1])  # Fri

# Unpacking — one variable per item
lat, lon = coordinates
print(f"Latitude: {lat},  Longitude: {lon}")
✅ Use a List when data will grow or change (shopping list, scores)
🔒 Use a Tuple when data is fixed (GPS, colours, weekdays)

Dictionaries

Dictionaries — Key → Value Pairs

Like a real dictionary: you look up a word (key) to get its meaning (value).

# Use curly braces {}, with  "key": value
student = {
    "name"       : "Alice",
    "age"        : 15,
    "city"       : "Kathmandu",
    "is_enrolled": True
}

# Access values using the KEY (not an index!)
print(student["name"])   # Alice
print(student["age"])    # 15
🔑 Keys are usually strings. Values can be ANY type — string, int, list, even another dictionary!

Dictionaries in Action 🛠️

student = {"name":"Alice", "age":15}

# Update a value
student["age"] = 16

# Add a new key
student["grade"] = "Grade 10"

# Delete a key
del student["age"]

# Explore keys & values
print(student.keys())
print(student.values())
REAL-WORLD EXAMPLE — PHONE BOOK
phone_book = {
    "Alice"  : "9841000001",
    "Bob"    : "9841000002"
}

name = input("Who to call? ")

if name in phone_book:
    print(phone_book[name])
else:
    print("Not found!")
Lists use [0],[1],[2]…  |  Dicts use ["name"],["age"]… Keys, not numbers!

List vs Tuple vs Dictionary — Quick Compare ⚡

📋 List
[1, 2, 3]
  • ✅Ordered
  • ✅Can add/remove items
  • ✅Access by index [0]
  • 🎯Use for: shopping lists, scores
🔒 Tuple
(1, 2, 3)
  • ✅Ordered
  • 🔒Cannot be changed
  • ✅Access by index [0]
  • 🎯Use for: GPS, RGB, weekdays
📖 Dictionary
{"k": v}
  • ✅Ordered (Python 3.7+)
  • ✅Can add/update/delete
  • 🔑Access by key ["name"]
  • 🎯Use for: profiles, phone books
💡 Rule of thumb: if items have labels → dict. If order matters and things change → list. If things never change → tuple.
