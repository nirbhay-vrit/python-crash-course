Returning More Than One Value

Separate values with commas after return. Python packs them into a tuple automatically — and you can unpack them on the other side!

def min_and_max(numbers):
    return min(numbers), max(numbers)  # ← tuple!

low, high = min_and_max([3, 9, 1, 7])  # unpack
print(low, high)                         # 1  9
return a, b → tuple
a, b = func() → unpack

Multiple Return — Patterns 🎁

Three values at once

def circle_stats(r):
    import math
    area  = math.pi * r ** 2
    diam  = 2 * r
    circ  = 2 * math.pi * r
    return area, diam, circ

a, d, c = circle_stats(5)
print(f"area={a:.2f} diam={d}")

The classic swap trick

a = 10
b = 20

# Swap in ONE line using tuple unpacking:
a, b = b, a

print(a, b)   # 20  10
No temp variable needed — Python packs the right side into a tuple first, then unpacks left.
💡 Use multiple return to group related results together. If you find yourself returning 5+ values, consider a dictionary or a class instead.

*args
Variable positional arguments

Accept ANY Number of Arguments

Prefix a parameter with * and Python collects all extra positional arguments into a tuple you can loop over.

def add_all(*args):
    return sum(args)      # args is a tuple

print(add_all(1, 2))          # 3
print(add_all(1, 2, 3))       # 6
print(add_all(10, 20, 30, 40)) # 100
* collects into a tuple
works with 0 args too

*args — Rules and Patterns ✳️

REGULAR PARAMS COME FIRST
def greet_all(greeting, *names):
    for name in names:
        print(f"{greeting}, {name}!")

greet_all("Hello", "Alice", "Bob", "Charlie")
# greeting = "Hello"
# names    = ("Alice", "Bob", "Charlie")
UNPACK A LIST WITH *
def add3(a, b, c):
    return a + b + c

nums = [10, 20, 30]
print(add3(*nums))   # 60
# * UNPACKS the list as 3 args
The * has two uses: collect into a tuple in def, and unpack a list in a call.

**kwargs
Variable keyword arguments

Accept ANY Named Arguments

Prefix a parameter with ** and Python collects all extra keyword arguments into a dictionary you can loop over or look up by key.

def print_info(**kwargs):
    for key, val in kwargs.items():
        print(f"  {key}: {val}")

print_info(name="Alice", age=30, city="Paris")
# name: Alice  |  age: 30  |  city: Paris
** collects into a dict
keys = argument names

① regular parameters
e.g. table
② *args

Composition

One Job Per Function

Big functions are hard to read and test. The fix: give each function one job, then write a coordinator that calls them in order.

❌ One giant function

def report(name, scores):
    total = 0
    for s in scores: total += s
    avg = total / len(scores)
    if avg >= 90: grade = "A"
    elif avg >= 80: grade = "B"
    ...
    print(name, avg, grade)
→

✅ Small helpers + coordinator

def get_average(scores): ...
def get_grade(avg): ...
def print_report(name, avg, grade): ...

def show_report(name, scores):
    avg   = get_average(scores)
    grade = get_grade(avg)
    print_report(name, avg, grade)

Why Small Functions Win 🔧

REUSABLE HELPERS
def calculate_average(scores):
    return sum(scores) / len(scores)

def get_grade(avg):
    if avg >= 90: return "A"
    elif avg >= 80: return "B"
    else: return "F"

# Test each helper alone!
print(calculate_average([100, 80]))  # 90
print(get_grade(75))               # B (wait—F!)
THE DRIVER PATTERN
def show_report(name, scores):
    avg   = calculate_average(scores)
    grade = get_grade(avg)
    print(f"{name}: {avg:.1f} ({grade})")

show_report("Alice", [92, 88, 95])
show_report("Bob",   [65, 72, 58])
Each helper has ONE job. The coordinator just orchestrates — no logic of its own.

Recursion
A function that calls itself

Solving a Problem with Itself

A recursive function solves a problem by calling itself on a smaller version of the same problem. Every recursive function needs two things:

① Base Case
The condition to STOP. Without it, the function runs forever!
② Recursive Case
Call itself with a smaller input, moving toward the base case.
def countdown(n):
    if n <= 0:              # BASE CASE — stop!
        print("Go!")
        return
    print(n)
    countdown(n - 1)         # RECURSIVE CASE — smaller n

Recursion — Classic Examples 🔁

Factorial: n! = n × (n-1)!

# 5! = 5 × 4! = 5 × 4 × 3! = ... = 120

def factorial(n):
    if n == 1:              # base case
        return 1
    return n * factorial(n - 1)

print(factorial(5))   # 120
print(factorial(3))   # 6
Each call reduces n by 1 until it hits 1.

Sum of a list

# sum([3,1,4]) = 3 + sum([1,4])
#              = 3 + 1 + sum([4])
#              = 3 + 1 + 4 + sum([]) = 8

def my_sum(numbers):
    if len(numbers) == 0:  # base case
        return 0
    return numbers[0] + my_sum(numbers[1:])

print(my_sum([3, 1, 4, 1, 5]))  # 14
⚠️ Always write the base case first. Without it, Python raises RecursionError: maximum recursion depth exceeded.
